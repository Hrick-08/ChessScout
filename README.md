## Features

- рҹҺҜ **Automatic Opponent Detection** вҖ” Displays a scouting card as soon as a game starts
- рҹ“Ҡ **Comprehensive Stats** вҖ” Rating history (Rapid, Blitz, Bullet), W/L/D record, accuracy, and more
- рҹ“ҡ **Opening Analysis** вҖ” Top 3 opening repertoire with win rates, separated by color
- рҹ”„ **Smart Caching** вҖ” Caches opponent data for 24 hours to reduce API calls
- вҡЎ **Fast Performance** вҖ” Parallel API fetches, progressive card rendering
- рҹҺЁ **Dark UI** вҖ” Matches chess.com's dark theme seamlessly
- рҹҡЁ **Smurf Detection** вҖ” Warns if opponent is a new account with high skill indicators [FUTURE SCOPE]

## How It Works

1. **Content Script** (`content.js`) injects into chess.com/game/* pages
2. **MutationObserver** detects when opponent username appears in the DOM
3. **API Module** (`api.js`) fetches profile, stats, and game history from chess.com's public API
4. **Compute Module** (`compute.js`) parses PGN data and computes statistics
5. **Card Module** builds the HTML UI and injects it into the page
6. **Cache** stores results in `chrome.storage.local` with 24-hour expiration

## Data Sources

All data comes from the **Chess.com Public API** (no authentication required):

- `/player/{username}` вҖ” Profile info (avatar, country, league, status)
- `/player/{username}/stats` вҖ” Current ratings and records
- `/player/{username}/games/archives` вҖ” List of available game archives
- `/player/{username}/games/{YYYY}/{MM}` вҖ” Games for each month

## Computed Statistics

### Opening Repertoire
- Parse opening names from PGN `[ECOUrl]` headers
- Separate by color (White/Black)
- Count frequency and calculate win rates
- Show top 3 per color

### Accuracy Trend
- Extract accuracy data from recent games
- Average last 20 games
- Detect improving/declining trend

### Record
- Aggregate W/L/D per time control (Rapid, Blitz, Bullet)
- Display as a proportional bar chart

### Winning/Losing Streak
- Find consecutive wins or losses from most recent games
- Display as "W3" or "L2" etc.

### Smurf Detection
- Flag if account age < 90 days AND (rating > 1400 OR accuracy > 85%)
- Show warning badge in card

## Performance Optimizations

- **Parallel fetching** вҖ” Profile + stats fetched concurrently
- **Rate limit handling** вҖ” 300ms delay between monthly archive requests
- **Progressive rendering** вҖ” Card shows immediately after profile load, game data fills in progressively
- **Smart caching** вҖ” Only re-fetch if cache expired or user manually refreshes
- **Silent error handling** вҖ” Missing months/data gracefully skipped

## Edge Cases Handled

- вң… Opponent has < 3 months of histories вҖ” Uses whatever exists
- вң… No accuracy data in games вҖ” Skips accuracy section
- вң… Opponent has 0 games in a time control вҖ” Shows "вҖ”" for rating
- вң… API rate limiting вҖ” Implements backoff and delays
- вң… Card rerender on rematch вҖ” Replaces old card with new one
- вң… Username not found вҖ” Error logged, card not shown

## API Rate Limiting

The Chess.com API has rate limits. The extension handles this by:

- **Caching for 24 hours** вҖ” Reduces repeated requests for same opponent
- **Staggering requests** вҖ” 300ms delay between monthly archive fetches
- **Silently skipping errors** вҖ” If a month 404s, continues without error

## Troubleshooting

### Card Not Showing

1. Ensure you're on a chess.com/game/* page
2. Wait for the game to load completely (look for opponent username)
3. Check extension permissions in Chrome extensions page
4. Open DevTools (F12) and check Console for errors

### Wrong Opponent Detected

The extension uses heuristics to find the opponent username. If detection fails:
1. Open DevTools Console (F12)
2. Look for `[Chess Scout]` logs
3. Report the issue with the DOM structure

### API Errors

If Chess.com API returns 404 or rate limit errors:
- Extension logs them to console
- Card rendering continues with available data
- Wait 24 hours for cache to expire and re-fetch
